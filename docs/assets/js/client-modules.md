# Client modules

## Purpose

The browser layer enhances the server-rendered dashboard with CRUD modals, global search, navigation, drag-and-drop sorting, context menus, visual updates, click tracking, password changes, and favicon refreshes.

## Location

- `assets/js/app.js`
- `assets/js/modules/*.js`
- `includes/templates/modals/*.php`
- `assets/css/main.css`, `assets/css/responsive.css`, and `assets/css/bookmark-colors.css`

## Inputs/Outputs

Inputs:

- Server-rendered element IDs, classes, and `data-*` attributes from `app/index.php` and the modal templates.
- Global configuration values such as `window.faviconConfig`, color maps, and bookmark background classes.
- JSON responses from `api/*.php`.
- Pointer, touch, keyboard, form, and resize events.

Outputs:

- DOM updates or page reloads after mutations.
- Flash messages for success and failure.
- Reorder requests generated by SortableJS.
- Debug messages controlled by `window.DEBUG`.

## Flow/Behavior

`assets/js/app.js` waits for `DOMContentLoaded` and loads scripts sequentially in this dependency order:

1. Flash messages and shared utilities.
2. Global search and page navigation.
3. Drag-and-drop and section management.
4. Modal management.
5. Bookmark, category, and page management.
6. Context menus, password management, favicon management, and click tracking.

Module responsibilities:

- `flash-messages.js` displays and dismisses user feedback.
- `utils.js` centralizes DOM updates, favicon rendering, color classes, and mobile-mode detection.
- `global-search.js` fetches all user bookmarks and provides debounced keyboard-accessible search.
- `page-navigation.js` switches pages and supports adjacent-page navigation.
- `drag-drop.js` persists bookmark and category ordering and disables unsuitable behavior in mobile mode.
- `section-management.js` handles expanded and collapsed category state. Categories with more than five bookmarks show the first six in full while collapsed, with a centered round toggle floating over the final row without covering its right-side edit action. Expanded categories add a small bottom buffer so the collapse toggle only partly overlaps the final link. On desktop, expanded categories float over the grid instead of reflowing later categories.
- `modal-management.js` opens, closes, and populates shared dialogs.
- `bookmark-management.js`, `category-management.js`, and `page-management.js` submit CRUD requests.
- `context-menu.js` provides empty-space and category-specific actions, including long-press support.
- `password-management.js` changes the password and coordinates logout behavior.
- `favicon-management.js` refreshes icons and applies fallback rendering.
- `click-tracking.js` reports bookmark activation and immediately applies the recent-use border state after a successful update. Dashboard, global-search, and open-all activations are tracked.

Debugging:

1. Run `DEBUG.enabled = true` in the browser console.
2. Optionally restrict output with `DEBUG.enableFor('MODAL')` or an array of module names.
3. Run `DEBUG.help()` for available commands.
4. Run `forceMobileMode()`, `forceDesktopMode()`, or `detectSimulationMode()` to inspect responsive drag-and-drop behavior after utilities load.

## Edge Cases/Failure Modes

- The modules are classic scripts that communicate through globals, so load order is part of their contract.
- When one module fails to load, then sequential loading stops and later modules are unavailable.
- The loader logs module-load failures to the console but does not display a user-facing recovery state.
- DOM identifiers and data attributes are shared contracts with PHP templates; renaming markup without updating the modules breaks behavior silently or at event time.
- Mobile detection combines viewport width, screen width, touch capability, and an optional forced mode. Browser device emulation can therefore behave differently from a physical device.
- Search results depend on a successful authenticated request to `get-all-bookmarks.php`.
